## 概述

我们为用户提供免费的电子邮件服务，用于发送和接收电子邮件。用户可以生成唯一的电子邮件地址并用它来发送和接收电子邮件。

在 https://kedaya.xyz/emails 开始使用电子邮件服务！


## 工作原理

我们使用 [resend](https://resend.com/) 来发送电子邮件。Resend 是一个免费的电子邮件服务，允许用户发送和接收电子邮件。

对于接收电子邮件，我们使用 [cloudflare](https://workers.cloudflare.com/) 电子邮件 worker 来接收和转发电子邮件。

它功能强大且稳定，而且使用简便。

## 有效期

除非您手动删除，否则电子邮件永远不会被删除。

一旦删除，将无法恢复。收件箱中的所有电子邮件将同时被删除。

## 限制

电子邮件地址的最大数量为 1000。

每个电子邮件地址可以接收无限制的电子邮件。

对于发送电子邮件，每天最多可发送 10 封电子邮件。

## API 参考

电子邮件 API 允许您创建和管理电子邮件地址，并检索收件箱中收到的电子邮件。

### 创建电子邮件地址

`POST /api/v1/email` 端点允许您创建新的电子邮件地址。

```bash
curl -X POST \
  -H "Content-Type: application/json" \
  -H "wrdo-api-key: YOUR_API_KEY" \
  -d '{
    "emailAddress": "your-suffix@kedaya.xyz"
  }' \
  https://kedaya.xyz/api/v1/email
```

#### 请求体 (参数)

```json
{
  "emailAddress": "your-suffix@kedaya.xyz" // 必需，后缀必须至少有5个字符
}
```

#### 授权头

- `wrdo-api-key`：您可以使用 API 密钥验证请求。
您可以在[账户设置](/dashboard/settings)中找到您的 API 密钥。
向请求添加头部 `wrdo-api-key: YOUR_API_KEY`。

#### 响应

成功时 (状态 201)：
```json
{
  id: string;
  userId: string;
  emailAddress: string;
  createdAt: Date;
  updatedAt: Date;
  deletedAt: Date | null;
}
```

#### 错误响应

- `401 未授权`：缺少或无效的 API 密钥
- `400 错误请求`：缺少电子邮件地址或无效的后缀（少于5个字符）
- `403 禁止`：已达到电子邮件地址配额
- `409 冲突`：电子邮件地址已存在
- `500 内部服务器错误`：服务器错误

### 获取电子邮件收件箱

`GET /api/v1/email/inbox` 端点允许您检索特定电子邮件地址的所有转发邮件。

<Callout type="warning" twClass="mt-0">
  在获取收件箱之前，您必须先创建电子邮件地址。
</Callout>

```bash
curl -X GET \
  -H "wrdo-api-key: YOUR_API_KEY" \
  "https://kedaya.xyz/api/v1/email/inbox?emailAddress=your-suffix@kedaya.xyz&page=1&size=10"
```

#### 查询参数

- `emailAddress`：要获取收件箱的电子邮件地址（必需）
- `page`：分页的页码（可选，默认：1）
- `size`：每页的电子邮件数量（可选，默认：10）

#### 授权头

- `wrdo-api-key`：您可以使用 API 密钥验证请求。
您可以在[账户设置](/dashboard/settings)中找到您的 API 密钥。
向请求添加头部 `wrdo-api-key: YOUR_API_KEY`。

#### 响应

成功时 (状态 200)：
```json
{
  "list": [
    {
      id: string
      from: string
      fromName: string
      to: string
      subject: string | null
      text: string | null
      html: string | null
      date: string | null
      messageId: string | null
      replyTo: string | null
      cc: string | null
      headers: string | null
      attachments: string | null
      readAt: Date | null
      createdAt: Date
      updatedAt: Date
    },
  ],
  "total": 25
}
```

#### 错误响应

- `401 未授权`：缺少或无效的 API 密钥
- `400 错误请求`：缺少 emailAddress 参数
- `404 未找到`：电子邮件地址未找到或已被删除
- `500 内部服务器错误`：服务器错误

### 删除电子邮件地址

`DELETE /api/v1/email` 端点允许您删除特定的电子邮件地址。

```bash
curl -X DELETE \
  -H "wrdo-api-key: YOUR_API_KEY" \
  -d '{
    "emailAddress": "your-suffix@kedaya.xyz"
  }' \
  "https://kedaya.xyz/api/v1/email"
```
